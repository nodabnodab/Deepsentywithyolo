# seugho
🐾 DeepSentry: YOLOv11 기반 유해 야생동물 피해 방지 및 실시간 모니터링 시스템
📖 프로젝트 개요 (About the Project)
DeepSentry는 농작물 피해의 주원인인 유해 야생동물을 실시간으로 탐지하고 모바일 애플리케이션을 통해 즉각적인 알림과 모니터링 기능을 제공하는 스마트 방재 시스템입니다.
기획 배경: 농업 현장에서 발생하는 야생동물 피해를 최소화하고, 관리자가 언제 어디서나 현장 상황을 파악할 수 있도록 돕기 위해 개발되었습니다.
수상 내역: 🏆 2025 제5회 전국 대학교 이그나이트 스토리텔링 경진대회 장려상 수상

🧑‍💻 나의 역할 (My Contribution)
이승호 (AI & Backend Developer) / 팀 규모: 2인
본 프로젝트에서 안드로이드(MAUI) UI/UX 화면 개발을 제외한 AI 모델 파이프라인 전체와 백엔드 통신 아키텍처를 전담했습니다.
AI 데이터 파이프라인 구축: 자체적인 데이터 수집, LabelMe 포맷 가공, YOLO 포맷 변환 등 데이터 전처리 과정 전담.
YOLOv11 커스텀 모델 구축 및 최적화: 객체 탐지 모델 학습, 검증 및 예측 로직 구현.
백엔드 통신 설계: 탐지된 객체의 로그 및 녹화 영상을 Firebase(Firestore & Storage)와 실시간으로 연동하는 비동기 통신 로직 구현.

(참고: 모바일 앱의 화면/UI 구성은 팀원이 담당하였습니다.)
✨ 주요 기능 및 특징 (Key Features)
1. 하드웨어의 제약 없는 유연한 AI 탐지 환경
초기 엣지 디바이스(Jetson) 도입을 고려했으나, 시스템의 유연성과 실용성을 극대화하기 위해 GPU가 탑재된 일반 PC 환경에서 구동되도록 최적화했습니다.
특정 카메라 장비에 종속되지 않으며 웹캠이나 일반 영상 파일 정보만 있으면 즉시 분석 및 구동이 가능합니다.

2. 타겟 맞춤형 객체 인식 (AI Object Detection)
농작물 피해의 주원인이 되는 곰, 멧돼지, 고라니 3종의 야생동물과 카메라에 가장 자주 포착되는 사람을 포함하여 총 4가지 핵심 클래스로 모델을 타겟팅하여 학습시켰습니다.
불필요한 클래스를 배제하여 탐지 효율과 정확도를 높였습니다.

3. 멀티스레딩 기반 비동기 처리
객체 탐지와 동시에 threading 라이브러리를 활용하여 영상 압축 및 업로드를 백그라운드에서 병렬로 처리합니다. 
2초 이상 감지된 객체는 자동으로 영상을 녹화하며 탐지 로그와 영상을 딜레이 없이 Firebase DB에 업로드합니다

4. PC와 모바일의 실시간 연동 (PC ↔ Firebase ↔ MAUI)
안드로이드 MAUI 앱과 PC 탐지 프로그램은 Firebase 계정을 매개로 강력하게 연동됩니다.
PC에서 감지된 정보(실시간 영상, 탐지 로그, 차트 통계)는 모바일 앱에서 즉시 확인 가능합니다.


🛠 기술 스택 (Tech Stack)
AI & Vision: Python, YOLOv11, OpenCV, Scikit-learn
Backend (Database): Firebase Firestore Database, Firebase Storage
Mobile App: C#, .NET MAUI
Tools & Environment: Visual Studio Code, Virtual Environments

🚀 실행 및 연동 가이드 (Getting Started)
DeepSentry 시스템은 복잡한 설치 과정 없이, PC 환경의 탐지 프로그램과 안드로이드 모바일 앱이 Firebase를 통해 클라우드로 연동되는 구조입니다.
PC (감지 및 처리부): GPU가 장착된 PC에서 Python 기반의 YOLOv11 탐지 프로그램을 실행합니다. (웹캠 또는 영상 파일 연결 필요)
Mobile (구동부): 안드로이드 기기에 MAUI로 빌드된 DeepSentry 앱을 설치합니다.
연동 (Firebase): PC 프로그램과 모바일 앱에 동일한 Firebase 프로젝트(API 및 계정 정보)가 세팅되어 있다면, PC에서 분석된 데이터가 자동으로 모바일 앱으로 실시간 동기화됩니다. 

💡 트러블슈팅 (Troubleshooting)
데이터베이스의 실시간성 확보: 초기 기획 단계에서는 가벼운 SQLite를 활용하여 로컬 DB를 구축했으나, PC-모바일 간 데이터 공유와 실시간 모니터링에 한계가 있었습니다. 
이를 해결하기 위해 클라우드 기반의 Firebase로 아키텍처를 전면 전환하여, 정형(로그)/비정형(영상) 데이터를 모바일 환경에 실시간으로 완벽하게 연동시키는 데 성공했습니다.
